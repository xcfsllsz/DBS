# DBMS内部结构

## 一、DBMS概述

> **DBMS（数据库管理系统）**：位于用户和操作系统之间的**系统软件**，用于管理数据库。

### DBMS的主要功能

| 功能 | 说明 |
|------|------|
| **数据定义** | DDL（Data Definition Language） |
| **数据操纵** | DML（Data Manipulation Language） |
| **数据控制** | DCL（Data Control Language） |
| **数据库运行管理** | 并发控制、安全性、完整性 |
| **数据库维护** | 备份、恢复、重组 |

```
┌─────────────────────────────────────────────────────┐
│                    DBMS                             │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐ │

│  └─────────────┘  └─────────────┘  └─────────────┘ │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐ │
│  │  模式映射   │  │  事务管理   │  │  并发控制   │ │

│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐ │

└─────────────────────────────────────────────────────┘
          ↓                         ↓
     用户视图                   存储结构
```

### 核心模块功能

| 模块 | 职责 |
|------|------|
| **DDL编译器** | 处理数据定义语言，创建模式 |
| **DML编译器** | 处理数据操作语言，执行查询 |
| **查询处理器** | 解析、优化、执行SQL |
| **模式映射** | 实现三级模式之间的转换 |
| **事务管理器** | 管理事务的ACID特性 |
| **并发控制器** | 处理多用户并发访问 |
| **日志管理器** | 记录数据库操作，支持恢复 |
| **存储管理器** | 管理磁盘空间和数据存取 |

## 三、数据字典

> **数据字典**：存放数据库**元数据**的系统表

### 数据字典内容

| 内容 | 说明 |
|------|------|
| 模式定义 | 各个模式的定义 |
| 完整性约束 | 主键、外键、约束条件 |
| 用户权限 | 用户账号、访问权限 |
| 审计信息 | 用户访问记录 |

### 数据字典作用

- 支持SQL语句编译
- 验证用户权限
- 优化查询执行

## 四、DBMS的查询处理

### 查询处理步骤

```
SQL语句
   ↓
1. 语法分析 → 解析树
   ↓
2. 语义检查 → 权限验证
   ↓
3. 查询优化 → 执行计划
   ↓
4. 执行 → 返回结果
```

### 查询优化类型

| 类型 | 说明 |
|------|------|
| **规则优化** | 基于启发式规则的优化 |
| **代价优化** | 基于统计信息的优化 |
| **语义优化** | 基于语义的等价变换 |

## 五、存储管理器

### 主要组件

| 组件 | 功能 |
|------|------|
| **文件管理器** | 管理磁盘空间分配 |
| **缓冲区管理器** | 管理内存缓存 |

```
CPU寄存器
    ↓

```

## 六、事务管理器

### 事务管理职责

1. **事务开始/结束**控制
2. **并发控制**
3. **故障恢复**
4. **日志管理**

### ACID特性

| 特性 | 说明 | 保证机制 |
|------|------|----------|
| **A**tomicity | 原子性 | 事务要么全做要么全不做 |
| **C**onsistency | 一致性 | 事务执行结果一致 |
| **I**solation | 隔离性 | 并发事务互不干扰 |
| **D**urability | 持久性 | 提交后数据不丢失 |

## 七、日志管理器

### 日志类型

| 类型 | 说明 |
|------|------|
| **undo日志** | 记录旧值，用于回滚 |
| **redo日志** | 记录新值，用于重做 |
| **undo/redo日志** | 记录两者 |

### 日志记录格式

```
<Start T>           - 事务开始
<T, X, old_v>       - undo日志
<T, X, new_v>       - redo日志
<T, X, new_v, old_v> - undo/redo日志
<Commit T>          - 事务提交
<Abort T>           - 事务中止
```

### 日志写入原则

> **先写日志原则(WAL)**：必须先把日志记录写入磁盘，然后才能写数据库

## 八、并发控制器

### 并发控制的目标

- 保证事务的**隔离性**
- 保证数据库的**一致性**
- 提高系统**吞吐量**

### 常用并发控制技术

| 技术 | 说明 |
|------|------|
| **封锁协议** | 使用锁来控制并发 |
| **时间戳** | 使用时间戳排序 |
| **乐观并发控制** | 假设冲突少，最后检查 |
| **多版本并发控制(MVCC)** | 维护多个版本 |

## 九、缓冲区管理器

### 缓冲区管理策略

| 策略 | 说明 |
|------|------|
| **LRU** | 最近最少使用 |
| **FIFO** | 先进先出 |
| **Clock** | 改进的LRU |

### 缓冲区相关操作

- **Pin**：将页固定在内存
- **Unpin**：释放页
- **Flush**：将脏页写回磁盘
