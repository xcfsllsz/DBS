# Data Definition（数据定义）

## 一、SQL概述

> **SQL（Structured Query Language）**：结构化查询语言，是关系数据库的标准语言。

### 1.1 SQL组成

| 组成部分 | 英文缩写 | 功能 |
|----------|----------|------|
| **数据定义** | DDL | 创建、修改、删除数据库对象 |
| **数据查询** | DQL | 查询数据库中的数据 |
| **数据操纵** | DML | 插入、更新、删除数据 |
| **数据控制** | DCL | 权限管理和事务控制 |

### 1.2 SQL特点

- **综合统一**：DDL、DML、DCL于一体
- **高度非过程化**：只需说明做什么，不需说明怎么做
- **面向集合**：操作对象是集合
- **简洁易学**：语法接近英语
- **统一标准**：ANSI SQL标准

## 二、数据类型

### 2.1 常用数据类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **INT/INTEGER** | 整数 | 1, 100, -5 |
| **FLOAT/DOUBLE** | 浮点数 | 3.14, -0.5 |
| **CHAR(n)** | 定长字符串 | '男', 'abc' |
| **VARCHAR(n)** | 变长字符串 | 'hello' |
| **DATE** | 日期 | '2024-01-01' |
| **DATETIME** | 日期时间 | '2024-01-01 12:00:00' |
| **BOOLEAN** | 布尔值 | TRUE, FALSE |

### 2.2 CHAR vs VARCHAR

| 类型 | 特点 |
|------|------|
| **CHAR(n)** | 定长，不足补空格；适合长度固定的数据 |
| **VARCHAR(n)** | 变长，按实际长度存储；适合长度变化的数据 |

## 三、数据库操作

### 3.1 创建数据库

```sql
CREATE DATABASE 数据库名;
```

### 3.2 删除数据库

```sql
DROP DATABASE 数据库名;
```

### 3.3 使用数据库

```sql
USE 数据库名;
```

## 四、表操作（DDL）

### 4.1 创建表

```sql
CREATE TABLE 表名 (
    列名1 数据类型 [约束],
    列名2 数据类型 [约束],
    ...
    [表级约束]
);
```

**示例**：

```sql
CREATE TABLE Student (
    SNo CHAR(10) PRIMARY KEY,          -- 主键约束
    SName VARCHAR(20) NOT NULL,        -- 非空约束
    Sage INT,
    Ssex CHAR(2),
    Sdept VARCHAR(20) DEFAULT 'CS'     -- 默认值
);
```

### 4.2 删除表

```sql
DROP TABLE 表名;
```

### 4.3 修改表结构

```sql
-- 添加列
ALTER TABLE 表名 ADD 列名 数据类型 [约束];

-- 删除列
ALTER TABLE 表名 DROP COLUMN 列名;

-- 修改列
ALTER TABLE 表名 MODIFY 列名 新数据类型;

-- 重命名列
ALTER TABLE 表名 CHANGE 旧列名 新列名 新数据类型;
```

## 五、约束(Constraint)

> **约束**：保证数据**完整性**和**一致性**的规则

### 5.1 约束类型

| 约束类型 | 关键字 | 说明 |
|----------|--------|------|
| **主键约束** | PRIMARY KEY | 唯一且非空 |
| **唯一约束** | UNIQUE | 值唯一（可有一个NULL） |
| **非空约束** | NOT NULL | 值不能为空 |
| **检查约束** | CHECK | 值满足条件 |
| **外键约束** | FOREIGN KEY | 引用其他表的主键 |
| **默认值约束** | DEFAULT | 默认值 |

### 5.2 约束详解

#### 主键约束(PRIMARY KEY)

```sql
-- 列级约束
CREATE TABLE Student (
    SNo CHAR(10) PRIMARY KEY
);

-- 表级约束
CREATE TABLE Student (
    SNo CHAR(10),
    PRIMARY KEY(SNo)
);

-- 复合主键
CREATE TABLE SC (
    SNo CHAR(10),
    CNo CHAR(10),
    PRIMARY KEY(SNo, CNo)  -- 复合主键
);
```

**特点**：
- 唯一标识表中每行
- 每个表**只能有一个**主键
- 主键列**不能为空**

#### 唯一约束(UNIQUE)

```sql
CREATE TABLE Student (
    SNo CHAR(10) PRIMARY KEY,
    IDCard CHAR(18) UNIQUE  -- 身份证号唯一
);
```

**特点**：
- 值唯一
- **可以有NULL值**（但只能有一个）
- 一个表可以有多个唯一约束

#### 非空约束(NOT NULL)

```sql
CREATE TABLE Student (
    SNo CHAR(10) PRIMARY KEY,
    SName VARCHAR(20) NOT NULL  -- 姓名不能为空
);
```

#### 外键约束(FOREIGN KEY)

```sql
CREATE TABLE SC (
    SNo CHAR(10),
    CNo CHAR(10),
    Grade INT,
    PRIMARY KEY(SNo, CNo),
    FOREIGN KEY(SNo) REFERENCES Student(SNo),
    FOREIGN KEY(CNo) REFERENCES Course(CNo)
);
```

**外键约束特点**：
- 引用**另一个表的主键**
- 被引用表称**父表**，引用表称**子表**
- 子表插入时，父表必须有对应记录
- 父表删除时，子表相关记录需先处理

#### 检查约束(CHECK)

```sql
CREATE TABLE Student (
    SNo CHAR(10) PRIMARY KEY,
    Sage INT CHECK (Sage >= 0 AND Sage <= 150),
    Ssex CHAR(2) CHECK (Ssex IN ('男', '女'))
);
```

#### 默认值约束(DEFAULT)

```sql
CREATE TABLE Student (
    SNo CHAR(10) PRIMARY KEY,
    Sdept VARCHAR(20) DEFAULT 'CS'
);
```

## 六、索引(Index)

> **索引**：加快数据检索的**数据结构**

### 6.1 创建索引

```sql
-- 单列索引
CREATE INDEX 索引名 ON 表名(列名);

-- 唯一索引
CREATE UNIQUE INDEX 索引名 ON 表名(列名);

-- 复合索引
CREATE INDEX 索引名 ON 表名(列名1, 列名2);
```

### 6.2 删除索引

```sql
DROP INDEX 索引名 ON 表名;
```

### 6.3 索引类型

| 类型 | 说明 |
|------|------|
| **B+树索引** | 最常用，适合范围查询 |
| **哈希索引** | 适合等值查询 |
| **唯一索引** | 索引列值唯一 |
| **全文索引** | 适合文本搜索 |

### 6.4 索引的优缺点

| 优点 | 缺点 |
|------|------|
| 加快查询速度 | 占用存储空间 |
| 加速表连接 | 降低更新速度 |
| 保证唯一性 | 需要维护索引 |

## 七、视图(View)

> **视图**：虚拟表，由查询结果组成

### 7.1 创建视图

```sql
CREATE VIEW 视图名 AS
SELECT 列1, 列2, ...
FROM 表名
WHERE 条件;
```

**示例**：

```sql
CREATE VIEW CS_Student AS
SELECT SNo, SName, Sage
FROM Student
WHERE Sdept = 'CS';
```

### 7.2 删除视图

```sql
DROP VIEW 视图名;
```

### 7.3 视图的特点

- **不存储数据**：只存储定义
- **简化查询**：封装复杂查询
- **数据安全**：限制用户访问范围
- **逻辑独立性**：改变表结构不影响应用

### 7.4 可更新视图条件

- 视图只从一个表导出
- 不包含聚合函数和DISTINCT
- 不包含GROUP BY或HAVING
- 不包含计算列

## 八、完整性规则

### 8.1 完整性类型

| 类型 | 说明 |
|------|------|
| **实体完整性** | 主键唯一且非空 |
| **参照完整性** | 外键引用必须存在 |
| **用户定义完整性** | 用户自定义约束 |
| **域完整性** | 列值符合定义域 |

### 8.2 完整性检查时机

| 时机 | 说明 |
|------|------|
| **INSERT** | 插入新记录时 |
| **UPDATE** | 更新记录时 |
| **DELETE** | 删除记录时 |

### SQL四大组成

| 组成 | 功能 | 关键字 |

| 约束 | 关键字 | 特点 |
|------|--------|------|
| 主键 | PRIMARY KEY | 唯一+非空，一个表一个 |
| 唯一 | UNIQUE | 唯一，可有一个NULL |
| 非空 | NOT NULL | 不能为空 |
| 外键 | FOREIGN KEY | 引用其他表主键 |
| 检查 | CHECK | 自定义条件 |
| 默认 | DEFAULT | 默认值 |

### CHAR vs VARCHAR

- **CHAR**：定长，效率高
- **VARCHAR**：变长，节省空间

### 索引优缺点

> **优点**：加快查询
>
> **缺点**：占用空间，降低更新速度

### 视图特点

- 不存储数据，只存储定义
- 简化复杂查询
- 提供数据安全
- 逻辑独立性
