# 各种数据库

## 一、数据库分类概述

> 数据库技术经历了从**层次/网状**到**关系型**再到**NoSQL/NewSQL**的发展历程。

### 1.1 数据库发展历程

```
文件系统 → 层次/网状数据库 → 关系数据库 → NoSQL/NewSQL
(1960s)    (1960-70s)        (1980s+)     (2000s+)
```

### 1.2 数据库类型

| 类型 | 代表产品 | 特点 |
|------|----------|------|
| **关系型** | MySQL, PostgreSQL, Oracle, SQL Server | 事务支持，ACID |
| **NoSQL** | MongoDB, Redis, Cassandra | 高扩展，灵活Schema |
| **NewSQL** | TiDB, CockroachDB | 兼顾关系型+NoSQL扩展性 |
| **时序数据库** | InfluxDB, TimescaleDB | 优化时间序列数据 |
| **图数据库** | Neo4j | 优化关系图数据 |

## 二、关系型数据库(RDBMS)

> **关系型数据库**：采用**关系模型**组织数据的数据库

### 2.1 关系模型核心概念

| 概念 | 说明 |
|------|------|
| **关系** | 二维表 |
| **元组** | 表的行 |
| **属性** | 表的列 |
| **主键** | 唯一标识 |
| **外键** | 表间关联 |

### 2.2 主流关系型数据库

| 数据库 | 特点 | 开发者 |
|--------|------|--------|
| **Oracle** | 企业级，功能最全面 | Oracle公司 |
| **MySQL** | 开源，应用最广泛 | Oracle公司 |
| **PostgreSQL** | 功能强大，可扩展 | 开源社区 |
| **SQL Server** | 微软企业级 | 微软 |
| **SQLite** | 轻量级，嵌入式 | 开源 |

### 2.3 MySQL特点

- **开源免费**
- **跨平台**
- **支持多种存储引擎**（InnoDB, MyISAM）
- **支持事务**（InnoDB引擎）
- **高并发读取**
- **易于学习和部署**

### 2.4 PostgreSQL特点

- **功能强大**：支持高级特性
- **可扩展**：支持自定义数据类型和函数
- **标准兼容**：严格遵循SQL标准
- **MVCC支持**：多版本并发控制
- **PostGIS**：地理信息系统支持

## 三、NoSQL数据库

> **NoSQL**：Not Only SQL，非关系型数据库

### 3.1 NoSQL类型分类

| 类型 | 数据模型 | 代表产品 | 特点 |
|------|----------|----------|------|
| **键值数据库** | Key-Value | Redis, Memcached | 极快，简单 |
| **文档数据库** | 文档(JSON) | MongoDB, CouchDB | 灵活Schema |
| **列族数据库** | 列族 | Cassandra, HBase | 列式存储 |
| **图数据库** | 图 | Neo4j, TigerGraph | 关系优化 |

### 3.2 键值数据库(Key-Value Store)

**代表：Redis**

| 特点 | 说明 |
|------|------|
| **高性能** | 内存存储，极快读写 |
| **丰富数据类型** | String, Hash, List, Set, ZSet |
| **持久化** | 支持RDB和AOF |
| **主从复制** | 高可用支持 |
| **应用场景** | 缓存、会话、队列 |

### 3.3 文档数据库(Document Store)

**代表：MongoDB**

| 特点 | 说明 |
|------|------|
| **Schema灵活** | 无固定结构 |
| **JSON存储** | BSON格式 |
| **索引支持** | 高效查询 |
| **聚合框架** | 数据分析 |
| **应用场景** | 内容管理，日志分析 |

**示例文档**：

```json
{
  "_id": "001",
  "name": "张三",
  "age": 20,
  "courses": ["数据库", "算法"],
  "address": {
    "city": "武汉",
    "street": "珞珈山"
  }
}
```

### 3.4 列族数据库(Column-Family Store)

**代表：Cassandra**

| 特点 | 说明 |
|------|------|
| **列式存储** | 列聚簇在一起 |
| **高可扩展** | 分布式架构 |
| **无单点故障** | P2P架构 |
| **写入优化** | 高写入吞吐 |
| **应用场景** | 日志分析，物联网 |

### 3.5 图数据库(Graph Database)

**代表：Neo4j**

| 特点 | 说明 |
|------|------|
| **节点和边** | 图结构存储 |
| **关系优先** | 关系存储在边 |
| **高效遍历** | 复杂关系查询 |
| **Cypher语言** | 图查询语言 |
| **应用场景** | 社交网络，推荐系统 |

## 四、NewSQL数据库

> **NewSQL**：兼具关系型和NoSQL优点的分布式数据库

### 4.1 NewSQL特点

| 特点 | 说明 |
|------|------|
| **SQL支持** | 完整SQL语法 |
| **ACID事务** | 强一致性 |
| **水平扩展** | 分布式架构 |
| **高性能** | 媲美NoSQL |

### 4.2 主流NewSQL产品

| 产品 | 特点 | 开发者 |
|------|------|--------|
| **TiDB** | MySQL兼容，水平扩展 | PingCAP |
| **CockroachDB** | PostgreSQL兼容，强一致 | CockroachDB公司 |
| **Google Spanner** | 全球分布，强一致 | Google |
| **ClustrixDB** | MySQL兼容 | Clustrix |

### 4.3 TiDB核心特性

- **MySQL兼容**：无需修改代码
- **水平扩展**：在线扩容缩容
- **强一致性**：分布式事务
- **高可用**：自动故障转移
- **实时HTAP**：OLTP + OLAP

## 五、时序数据库(TSDB)

> **时序数据库**：专门存储**时间序列数据**

### 5.1 时序数据特点

| 特点 | 说明 |
|------|------|
| **时间戳** | 数据带有时间属性 |
| **持续写入** | 数据持续产生 |
| **数据量大** | 积累快 |
| **很少更新** | 历史数据一般不变 |
| **趋势分析** | 关注聚合查询 |

### 5.2 主流时序数据库

| 产品 | 特点 |
|------|------|
| **InfluxDB** | 最流行，InfluxQL |
| **TimescaleDB** | PostgreSQL扩展 |
| **Prometheus** | 监控领域，Pull模式 |
| **OpenTSDB** | HBase后端 |

### 5.3 应用场景

- **监控告警**：服务器指标
- **IoT物联网**：传感器数据
- **金融行情**：股票价格
- **日志分析**：应用日志

## 七、数据库选型指南

### 7.1 选型考虑因素

| 因素 | 说明 |
|------|------|
| **数据模型** | 结构化/半结构化/非结构化 |
| **一致性要求** | 强一致/最终一致 |
| **扩展需求** | 垂直扩展/水平扩展 |
| **事务需求** | 是否需要ACID |
| **性能要求** | 读写比例，吞吐量 |
| **运维成本** | 团队技术栈 |

### 7.2 选型建议

| 场景 | 推荐类型 |
|------|----------|
| 传统业务系统 | 关系型(MySQL, PostgreSQL) |
| 缓存系统 | Redis |
| 内容管理 | MongoDB |
| 社交网络 | 图数据库 |
| 监控时序 | 时序数据库 |
| 大数据平台 | 列族数据库 |
| 高并发+强一致 | NewSQL |

## 考点速记

### 数据库分类

| 类型 | 代表 | 特点 |
|------|------|------|
| 关系型 | MySQL, PostgreSQL | ACID, SQL |
| 键值 | Redis | 高性能，简单 |
| 文档 | MongoDB | 灵活Schema |
| 列族 | Cassandra | 列式存储 |
| 图 | Neo4j | 关系优化 |
| NewSQL | TiDB | 关系+扩展 |
| 时序 | InfluxDB | 时间序列 |

### MySQL存储引擎

| 引擎 | 特点 |
|------|------|
| InnoDB | 支持事务，行级锁 |
| MyISAM | 不支持事务，表级锁 |

### NoSQL vs SQL

| 对比 | SQL | NoSQL |
|------|-----|-------|
| 数据模型 | 关系 | 多样 |
| Schema | 固定 | 灵活 |
| 事务 | ACID | 弱事务 |
| 扩展 | 垂直 | 水平 |
| 查询 | SQL | 各自API |

### CAP理论（重点）

> **CAP**：分布式系统只能同时满足其中两个
>
> - **C**onsistency：一致性
> - **A**vailability：可用性
> - **P**artition Tolerance：分区容错性

### BASE理论

> **BASE**：最终一致性的妥协
>
> - **B**asically **A**vailable：基本可用
> - **S**oft State：软状态
> - **E**ventually Consistent：最终一致
