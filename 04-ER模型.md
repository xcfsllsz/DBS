# Entity-Relationship Models（实体联系模型）

## 一、ER模型概述

> **ER模型（Entity-Relationship Model）**：用于描述**现实世界**的概念模型，是数据库设计的重要工具。

### 1.1 核心概念

| 概念 | 英文 | 说明 |
|------|------|------|
| **实体** | Entity | 客观存在可区别的事物 |
| **属性** | Attribute | 实体的特征 |
| **联系** | Relationship | 实体之间的关联 |

### 1.2 ER图基本符号

```
┌─────────────────┐
│     实体         │  ← 矩形
│    Student      │
└─────────────────┘

┌─────────────────┐
│    联系         │  ← 菱形
│   Enroll        │
└─────────────────┘

┌─────────────────┐
●────── 属性 ─────●  ← 椭圆
   SName
```

## 二、实体(Entity)

### 2.1 实体的定义

> **实体**：客观存在且可**相互区分**的事物

### 2.2 实体类型与实体集

| 概念 | 说明 |
|------|------|
| **实体型** | 用实体名和属性名集合表示，如：Student(SNo, SName, Sage) |
| **实体值** | 具体的实体，如：S001, 张三, 20 |
| **实体集** | 同类型实体的集合 |

### 2.3 实体类型特征

- 具有**关键码**（Key）：能唯一标识实体的属性
- 用**矩形**表示
- 必须有**属性描述**

## 三、属性(Attribute)

### 3.1 属性的定义

> **属性**：实体所具有的**特征或性质**

### 3.2 属性类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **简单属性** | 不可再分的属性 | 年龄、性别 |
| **复合属性** | 可再分为多个属性 | 地址（省、市、区） |
| **单值属性** | 一个值 | 学号 |
| **多值属性** | 多个值 | 电话（可能有多个） |
| **派生属性** | 可由其他属性计算得出 | 年龄（由出生日期计算） |
| **存储属性** | 实际存储的属性 | 出生日期 |

### 3.3 属性的图形表示

```
┌──────────────┐
│    实体      │
│ ┌──────────┐ │
│ │  ○ SNo   │ │  ○ = 简单属性
│ │  ● SName │ │  ● = 复合属性
│ │ ○○ Phone │ │ ○○ = 多值属性（双椭圆）
│ │ () Age   │ │ () = 派生属性（虚线）
│ └──────────┘ │
└──────────────┘
```

### 3.4 关键码(Key)

> **超键(Super Key)**：能唯一标识实体的属性集
>
> **候选键(Candidate Key)**：最小的超键
>
> **主键(Primary Key)**：选定的候选键
>
> **主属性**：包含在候选键中的属性
>
> **非主属性**：不包含在任何候选键中的属性

**示例**：
- {学号, 姓名} → 超键
- {学号} → 候选键/主键
- {学号} → 主属性
- {姓名, 系别} → 非主属性

## 四、联系(Relationship)

### 4.1 联系的定义

> **联系**：实体之间的**相互关联**

### 4.2 联系的类型

#### 参与方式

| 方式 | 说明 | 图示 |
|------|------|------|
| **完全参与** | 实体的每个实例都参与联系 | 双线 |
| **部分参与** | 只有部分实体实例参与 | 单线 |

#### 基数(Cardinality)

| 类型 | 说明 | 示例 |
|------|------|------|
| **一对一(1:1)** | 每个实体只对应一个联系 | 校长-学校 |
| **一对多(1:N)** | 一个实体对应多个 | 班级-学生 |
| **多对一(N:1)** | 多个实体对应一个 | 学生-班级 |
| **多对多(M:N)** | 多个实体对应多个 | 学生-课程 |

#### 联系类型的判定

| 实体A | 实体B | 联系类型 |
|-------|-------|----------|
| 1 | 1 | 1:1 |
| 1 | N | 1:N |
| N | 1 | N:1 |
| N | M | M:N |

### 4.3 联系集的属性

> **联系集的属性**：当联系为M:N时，可以有自己的属性

**示例**：
```
Student(SNo, SName)
Course(CNo, CName)
SC(SNo, CNo, Grade)  ← 联系集SC有自己的属性Grade
```

### 4.4 联系与约束

| 约束类型 | 说明 |
|----------|------|
| **映射基数** | 1, N, M的具体数量 |
| **参与约束** | 完全参与/部分参与 |
| **存在依赖** | 实体是否存在依赖另一实体 |

## 五、ER图绘制

### 5.1 基本规则

```
矩形        → 实体集
椭圆        → 属性
菱形        → 联系
双椭圆      → 多值属性
虚线椭圆    → 派生属性
双线        → 完全参与
双矩形      → 弱实体集
```

### 5.2 完整ER图示例

```
         ┌───────────────────────────────┐
         │         Student               │
         │  ● SNo (PK)                   │
         │  ○ SName                      │
         │  ○ Sage                       │
         │  ○○ SPhone                    │
         └──────────┬────────────────────┘
                    │
                    │ M:N (Enroll)
                    │ has
                    │
         ┌──────────┴────────────────────┐
         │         SC                    │
         │     (Grade) ← 联系属性        │
         └──────────┬────────────────────┘
                    │
                    │ 1:N
                    │ belong to
         ┌──────────┴────────────────────┐
         │         Course               │
         │  ● CNo (PK)                  │
         │  ○ CName                     │
         │  ○ Credit                    │
         └───────────────────────────────┘
```

## 六、弱实体集

### 6.1 弱实体集定义

> **弱实体集**：没有足够的属性形成主键的实体集

### 6.2 弱实体集特征

- **依赖强实体集**存在
- 用**双矩形**表示
- 联系用**双菱形**表示
- 需要**拥有者实体集的主键**作为外键

### 6.3 弱实体集示例

```
┌─────────────────┐          ┌─────────────────┐
│    Department   │ 1    N  │     Course      │
│                 │◄────────│                 │
│  ● DNo (PK)    │  own     │  ○ CNo          │
│  ○ DName        │         │  ○ CName        │
└─────────────────┘          └─────────────────┘
                                   ▲
                                   │ identifying
                                   │ relationship
                                   │ (双菱形)
                                   │
                            ┌──────┴──────┐
                            │   无         │
                            └─────────────┘
```

## 七、ER模型到关系模型的转换

### 7.1 转换规则

| ER组件 | 转换结果 |
|--------|----------|
| 实体集 | 关系（表） |
| 属性 | 列 |
| 主键 | 主键约束 |
| 联系(1:1) | 外键 或 合并 |
| 联系(1:N) | 外键 |
| 联系(M:N) | 新关系 |
| 弱实体集 | 包含外键的新关系 |

### 7.2 转换示例

**实体集转换**：
```
Student(SNo, SName, Sage)  →  CREATE TABLE Student(...)
```

**1:1联系转换**：
```
Employee --- Manage --- Department
↓
在Employee或Department中添加外键
```

**1:N联系转换**：
```
Department 1───N Employee
↓
在Employee中添加外键指向Department
```

**M:N联系转换**：
```
Student M───N Course
↓
创建新关系SC(SNo, CNo, Grade)
```

## 考点速记

### ER模型三要素

| 要素 | 说明 |
|------|------|
| **实体** | 客观存在可区分的事物 |
| **属性** | 实体的特征 |
| **联系** | 实体间的关联 |

### 属性类型

- **简单属性** vs **复合属性**
- **单值属性** vs **多值属性**
- **派生属性** vs **存储属性**

### 关键码

> **超键** → 能唯一标识
>
> **候选键** → 最小的超键
>
> **主键** → 选定的候选键

### 联系类型

| 类型 | 说明 | 转换 |
|------|------|------|
| 1:1 | 一对一 | 外键 |
| 1:N | 一对多 | 外键 |
| M:N | 多对多 | 新关系 |

### ER图符号

```
矩形 = 实体集
椭圆 = 属性
菱形 = 联系
双椭圆 = 多值属性
虚线椭圆 = 派生属性
双矩形 = 弱实体集
双线 = 完全参与
```
