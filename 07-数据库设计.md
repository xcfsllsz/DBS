# 数据库设计

## 一、数据库设计概述

> **数据库设计**：根据应用需求，构建**高效**、**规范**的数据库模式

### 1.1 设计目标

| 目标 | 说明 |
|------|------|
| **功能完整** | 满足所有业务需求 |
| **性能优良** | 查询效率高 |
| **结构合理** | 规范化，无冗余 |
| **维护方便** | 易于扩展和维护 |

### 1.2 设计步骤

```
┌─────────────┐
│  需求分析   │  ← 了解用户需求
└──────┬──────┘
       ↓
┌──────┴──────┐
│  概念设计   │  ← ER模型
└──────┬──────┘
       ↓
┌──────┴──────┐
│  逻辑设计   │  ← 关系模型
└──────┬──────┘
       ↓
┌──────┴──────┐
│  物理设计   │  ← 存储结构
└─────────────┘
```

## 三、概念设计

> **概念设计**：建立概念数据模型，通常使用**ER模型**

### 3.1 设计策略

| 策略 | 说明 |
|------|------|
| **自顶向下** | 从全局开始，逐步细化 |
| **自底向上** | 从局部开始，整合全局 |
| **混合策略** | 两者结合 |

### 3.2 概念设计步骤

```
1. 确定实体集
2. 确定联系集
3. 确定属性
4. 确定键
5. 绘制ER图
6. 优化ER图
```

### 3.3 ER图设计原则

| 原则 | 说明 |
|------|------|
| **一致性** | 实体、联系不矛盾 |
| **最小冗余** | 避免重复存储 |
| **自然性** | 符合用户认知 |
| **最小性** | 实体集最小化 |

## 四、逻辑设计

> **逻辑设计**：将ER模型转换为**关系模型**

### 4.1 转换规则

| ER组件 | 转换结果 |
|--------|----------|
| 实体集 | 关系（表） |
| 属性 | 列 |
| 键属性 | 主键 |
| 联系(1:1) | 外键或合并 |
| 联系(1:N) | 外键 |
| 联系(M:N) | 新关系 |
| 弱实体集 | 含外键的新关系 |

### 4.2 关系模式优化

**合并关系**（1:1联系）：
```sql
-- 方式1：在A表中添加外键指向B表
-- 方式2：合并A和B为一个关系
```

**外键实现**（1:N联系）：
```sql
-- 在N端实体表中添加外键
CREATE TABLE Employee (
    ENo CHAR(10) PRIMARY KEY,
    EName VARCHAR(20),
    DNo CHAR(10) REFERENCES Department(DNo)
);
```

**新关系实现**（M:N联系）：
```sql
-- 创建新关系，包含两个实体的主键
CREATE TABLE SC (
    SNo CHAR(10),
    CNo CHAR(10),
    Grade INT,
    PRIMARY KEY(SNo, CNo),
    FOREIGN KEY(SNo) REFERENCES Student(SNo),
    FOREIGN KEY(CNo) REFERENCES Course(CNo)
);
```

### 4.3 M:N联系的转换

> **M:N联系**必须转换为**新关系**

**步骤**：
1. 创建新关系R
2. 包含两个实体的主键作为外键
3. 如有属性，一并加入
4. 组合主键作为R的主键

## 五、规范化理论

### 5.1 范式(Normal Form)概述

| 范式 | 英文 | 要求 |
|------|------|------|
| **1NF** | 第一范式 | 属性不可再分 |
| **2NF** | 第二范式 | 1NF + 无部分依赖 |
| **3NF** | 第三范式 | 2NF + 无传递依赖 |
| **BCNF** | 巴斯-科德范式 | 每个决定因素是候选键 |
| **4NF** | 第四范式 | 没有多值依赖 |
| **5NF** | 第五范式 | 无连接依赖 |

### 5.2 第一范式(1NF)

> **定义**：属性不可再分

**违反1NF示例**：
```
学生(学号, 姓名, 联系方式)
-- 联系方式包含电话、邮箱 → 可再分
```

**符合1NF**：
```
学生(学号, 姓名, 电话, 邮箱)
```

### 5.3 第二范式(2NF)

> **定义**：在1NF基础上，非主属性**完全函数依赖**于主键

**部分依赖**（违反2NF）：
```
SC(学号, 课程号, 课程名, 成绩)
-- 主键：(学号, 课程号)
-- 课程名只依赖于课程号 → 部分依赖
```

**符合2NF**：
```
Student(学号, 姓名, ...)
Course(课程号, 课程名, ...)
SC(学号, 课程号, 成绩)
```

### 5.4 第三范式(3NF)

> **定义**：在2NF基础上，非主属性**不传递依赖**于主键

**传递依赖**（违反3NF）：
```
Student(学号, 姓名, 系号, 系名)
-- 主键：学号
-- 系号 → 系名，而学号 → 系号
-- 系名传递依赖于学号
```

**符合3NF**：
```
Student(学号, 姓名, 系号)
Department(系号, 系名)
```

### 5.5 BCNF范式

> **定义**：每一个决定因素都包含**候选键**

**违反BCNF示例**：
```
Course_Teacher(课程号, 教师号, 教师名)
-- 候选键：(课程号, 教师号) 或 (课程号, 教师名)
-- 但教师号 → 教师名，教师号不是候选键
```

**符合BCNF**：
```
Course(课程号, 课程名)
Teacher(教师号, 教师名)
Course_Teacher(课程号, 教师号)
```

### 5.6 范式对比

```
1NF ⊂ 2NF ⊂ 3NF ⊂ BCNF ⊂ 4NF ⊂ 5NF
 ↑                                  ↑
最低                                 最高
规范化程度                          规范化程度
```

## 六、函数依赖

### 6.1 函数依赖定义

> **函数依赖**：X → Y，X决定Y，Y函数依赖于X

| 类型 | 定义 |
|------|------|
| **完全函数依赖** | X → Y，X的任一真子集不能决定Y |
| **部分函数依赖** | X → Y，但X的真子集也能决定Y |
| **传递函数依赖** | X → Y, Y → Z，则X → Z |

### 6.2 Armstrong公理

| 公理 | 规则 |
|------|------|
| **自反律** | 若Y⊆X，则X → Y |
| **增广律** | 若X → Y，则XZ → YZ |
| **传递律** | 若X → Y, Y → Z，则X → Z |

### 6.3 派生规则

| 规则 | 说明 |
|------|------|
| **合并规则** | X → Y, X → Z ⇒ X → YZ |
| **分解规则** | X → YZ ⇒ X → Y, X → Z |
| **伪传递规则** | X → Y, WY → Z ⇒ XW → Z |

## 七、反规范化

> **反规范化**：为提高性能，适当增加冗余

### 7.1 反规范化时机

| 时机 | 原因 |
|------|------|
| 查询频繁 | 减少连接操作 |
| 实时性要求高 | 避免实时计算 |
| 硬件限制 | 无法水平扩展 |

### 7.2 反规范化方法

| 方法 | 说明 |
|------|------|
| **增加冗余列** | 预计算并存储 |
| **增加派生列** | 存储汇总数据 |
| **表合并** | 减少连接 |
| **表分割** | 水平/垂直分割 |

### 7.3 反规范化风险

- **数据不一致**
- **更新复杂**
- **存储增加**

## 八、数据库设计案例

### 8.1 学生选课系统

**ER图设计**：
```
Student(SNo, SName, Sage, Sdept)
Course(CNo, CName, Credit, TNo)
SC(SNo, CNo, Grade)
Teacher(TNo, TName)
```

**关系模型**：
```sql
Student(SNo PK, SName, Sage, Sdept)
Course(CNo PK, CName, Credit, TNo FK)
SC(SNo FK, CNo FK, Grade, PRIMARY KEY(SNo, CNo))
Teacher(TNo PK, TName)
```

### 8.2 教务管理系统

**主要实体**：
- 学生(Student)
- 课程(Course)
- 教师(Teacher)
- 班级(Class)
- 系部(Department)

**主要联系**：
- 学生-班级：N:1
- 班级-系部：N:1
- 教师-系部：N:1
- 学生-课程：M:N (SC)
- 教师-课程：1:N
- 课程-先修课：自关联1:N
