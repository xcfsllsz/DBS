# 三级映射模式

## 一、三级模式体系结构

> 数据库系统的**三级模式结构**是数据库系统内部的核心架构，定义了数据抽象和独立性的层次。

### 1.1 三级模式定义

| 模式 | 名称 | 别名 | 说明 | 数量 |
|------|------|------|------|------|
| **外模式** | External Schema | 子模式/用户模式 | 用户视角看到的数据库 | 多个 |
| **概念模式** | Conceptual Schema | 逻辑模式 | 整个数据库的全局逻辑结构 | 唯一 |
| **内模式** | Internal Schema | 存储模式 | 数据物理存储结构 | 唯一 |

### 1.2 三级模式详解

#### 外模式（用户模式）

- **定义**：数据库用户能够看到和使用的**局部**数据
- **特点**：
  - 每个用户可以有**多个**外模式
  - 是数据库系统的**用户视图**
  - 定义用户对数据的操作权限

#### 概念模式（逻辑模式）

- **定义**：整个数据库的**全局逻辑结构**
- **特点**：
  - 数据库**只有一个**概念模式
  - 定义数据库的**逻辑结构**（关系、属性、约束）
  - 不涉及物理存储细节

#### 内模式（存储模式）

- **定义**：数据在数据库中的**物理存储结构**
- **特点**：
  - 数据库**只有一个**内模式
  - 定义数据的**存储方式**、索引、存取路径
  - 与硬件平台相关

## 二、两级映射

> **映射**：实现三级模式之间转换的机制

### 2.1 映射类型

```
┌─────────────┐
│   外模式    │ ← 用户视图
└──────┬──────┘
       │ 外模式/概念模式映射
       ↓
┌──────┴──────┐
│  概念模式   │ ← 全局逻辑
└──────┬──────┘
       │ 概念模式/内模式映射
       ↓
┌──────┴──────┐
│   内模式    │ ← 物理存储
└─────────────┘
```

| 映射 | 功能 |
|------|------|
| **外模式/概念模式映射** | 把用户视图转换为全局逻辑结构 |
| **概念模式/内模式映射** | 把逻辑结构映射为物理存储 |

### 2.2 映射的作用

1. **数据独立性**：通过映射实现
   - **逻辑独立性**：概念模式变化时，外模式不变
   - **物理独立性**：内模式变化时，概念模式不变

2. **数据抽象**：隐藏实现细节

## 三、数据独立性

> **数据独立性**：应用程序与数据库的物理/逻辑存储结构相互独立

### 3.1 逻辑独立性

| 项目 | 说明 |
|------|------|
| **定义** | 概念模式改变时，外模式不变 |
| **原因** | 外模式/概念模式映射 |
| **影响** | 应用程序不需要改变 |

**举例**：
- 增加新字段 → 概念模式变，外模式不变
- 修改字段名 → 修改映射即可

### 3.2 物理独立性

| 项目 | 说明 |
|------|------|
| **定义** | 内模式改变时，概念模式不变 |
| **原因** | 概念模式/内模式映射 |
| **影响** | 应用程序不需要改变 |

**举例**：
- 改变存储结构 → 只需修改映射
- 建立新索引 → 物理变化，逻辑不变

### 3.3 数据独立性对比

| 类型 | 变化内容 | 映射保障 |
|------|----------|----------|
| **逻辑独立性** | 概念模式（全局逻辑） | 外模式/概念模式映射 |
| **物理独立性** | 内模式（存储结构） | 概念模式/内模式映射 |

## 四、数据抽象

> 数据库系统通过**三级模式**实现数据抽象

### 4.1 抽象层次

| 层次 | 说明 | 类比 |
|------|------|------|
| **用户级** | 外模式 | 局部视图 |
| **全局级** | 概念模式 | 整个社区的规划图 |
| **存储级** | 内模式 | 实际施工图 |

### 4.2 物理数据独立性

- 定义：数据库的**物理模式**改变时，全局逻辑不变
- 原因：物理存储的变化对应用程序透明
- 体现：DBMS的核心目标之一

## 五、三级模式的实际应用

### 5.1 模式定义语言（DDL）

| DDL类型 | 说明 |
|---------|------|
| 外模式DDL | 定义外模式 |
| 概念模式DDL | 定义概念模式 |
| 内模式DDL | 定义内模式 |

### 5.2 模式转换过程

```
外模式定义 → 外模式
     ↓
概念模式定义 → 概念模式
     ↓
内模式定义 → 内模式
```

### 5.3 两级映射的实例

**例：用户查询"学生信息"**

1. 用户使用**外模式**（SELECT语句）
2. **外模式/概念模式映射**转换查询
3. **概念模式/内模式映射**找到物理位置
4. 从**内模式**（磁盘）读取数据

## 考点速记

### 三级模式核心要点

| 模式 | 名称 | 数量 | 内容 |
|------|------|------|------|
| 外模式 | 用户模式 | **多个** | 用户局部数据视图 |
| 概念模式 | 逻辑模式 | **唯一** | 全局逻辑结构 |
| 内模式 | 存储模式 | **唯一** | 物理存储结构 |

### 两级映射

| 映射 | 作用 |
|------|------|
| 外模式/概念模式映射 | 连接用户视图和全局逻辑 |
| 概念模式/内模式映射 | 连接逻辑结构和物理存储 |

### 数据独立性（重点）

> **逻辑独立性**：概念模式变化 → 外模式不变 → 外模式/概念模式映射保护
>
> **物理独立性**：内模式变化 → 概念模式不变 → 概念模式/内模式映射保护

### 必背概念

1. **外模式**：用户能看到的数据库部分，**可有多个**
2. **概念模式**：整个数据库的全局逻辑，**只有一个**
3. **内模式**：数据物理存储结构，**只有一个**
4. **映射**：实现模式间转换，保证数据独立性
