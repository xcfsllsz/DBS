# 事务、故障与并发控制

## 一、事务(Transaction)

### 1.1 事务的概念

> **事务**：用户定义的一组数据库操作，**要么全做要么全不做**，是不可分割的工作单位

### 1.2 事务定义语句

```sql
BEGIN TRANSACTION;  -- 事务开始
SQL语句...
COMMIT;             -- 提交，永久保存
ROLLBACK;           -- 回滚，撤销
```

### 1.3 事务的特性（ACID）

| 特性 | 英文 | 说明 | 记忆口诀 |
|------|------|------|----------|
| **原子性** | Atomicity | 全做或全不做 | 不可分 |
| **一致性** | Consistency | 执行前后数据一致 | 守恒 |
| **隔离性** | Isolation | 并发执行互不干扰 | 独立 |
| **持久性** | Durability | 提交后不丢失 | 永久 |

### 1.4 ACID详解

**原子性**：
```sql
-- 转账操作
BEGIN TRANSACTION;
UPDATE Account SET Balance=Balance-1000 WHERE ANO='A';
UPDATE Account SET Balance=Balance+1000 WHERE ANO='B';
COMMIT;
-- 要么全部成功，要么全部失败
```

**一致性**：
- 事务执行前后，**数据库状态一致**
- 例如：转账前后，双方总额不变

**隔离性**：
- 并发事务之间**相互隔离**
- 中间结果对其他事务不可见

**持久性**：
- 提交后，结果**永久保存**
- 即使系统故障也不丢失

## 二、事务状态

```
         ┌─────────┐
         │  活动   │ ← 初始状态
         └────┬────┘
              │
    ┌─────────┴─────────┐
    ↓                   ↓
┌────┴────┐        ┌────┴────┐
│ 部分提交 │        │  失败   │
└────┬────┘        └────┬────┘
     ↓                   ↓
┌────┴────┐        ┌────┴────┐
│  提交   │        │  中止   │
└─────────┘        └────┬────┘
                         ↓
                    ┌────┴────┐
                    │  回滚   │
                    └─────────┘
```

| 状态 | 说明 |
|------|------|
| **活动** | 正在执行 |
| **部分提交** | 执行完成，结果在内存 |
| **提交** | 成功写入磁盘 |
| **失败** | 无法继续执行 |
| **中止** | 回滚后回到初始状态 |

## 三、故障类型

### 3.1 故障分类

| 类型 | 说明 | 影响 |
|------|------|------|
| **事务故障** | 事务内部错误 | 单个事务 |
| **系统故障** | 软故障（停电等） | 所有活动事务 |
| **介质故障** | 硬故障（磁盘损坏） | 整个数据库 |
| **计算机病毒** | 人为破坏 | 不确定 |

### 3.2 故障恢复

| 故障类型 | 恢复方法 |
|----------|----------|
| **事务故障** | UNDO（回滚） |
| **系统故障** | UNDO + REDO |
| **介质故障** | 装入备份 + REDO |

## 四、并发控制

### 4.1 并发控制的目的

| 目的 | 说明 |
|------|------|
| **提高吞吐量** | 多事务并行执行 |
| **提高资源利用率** | CPU和磁盘并行 |
| **减少等待时间** | 减少事务阻塞 |

### 4.2 并发执行的问题

> **并发执行**可能引发以下问题：

| 问题 | 说明 | 示例 |
|------|------|------|
| **丢失更新** | 两个事务修改同一数据 | T1,T2都读X=10，T1写X=11，T2也写X=11 |
| **脏读** | 读取未提交数据 | T1修改X=11，T2读X=11，T1回滚 |
| **不可重复读** | 同一事务两次读取不同 | T1两次读X，第一次10，第二次被T2改为11 |
| **幻读** | 同一事务两次结果集不同 | T1两次统计人数，第二次T2插入了新记录 |

### 4.3 数据不一致问题详解

**丢失更新（Lost Update）**：
```
T1: R(X) → X=X+100 → W(X)
T2:          R(X) → X=X-50 → W(X)
-- T1的更新被T2覆盖
```

**脏读（Dirty Read）**：
```
T1: W(X) → COMMIT/Rollback
T2:       R(X)  ← 读到T1未提交的修改
```

**不可重复读（Non-repeatable Read）**：
```
T1: R(X) → R(X) → 比较两次X是否相同
T2:       W(X) → COMMIT
```

**幻读（Phantom Read）**：
```
T1: SELECT COUNT(*) → 结果1
T2: INSERT → COMMIT
T1: SELECT COUNT(*) → 结果2
```

## 五、封锁协议

> **封锁**：对数据对象加锁，控制并发

### 5.1 锁类型

| 锁类型 | 英文 | 说明 | 兼容性 |
|--------|------|------|--------|
| **排他锁(X锁)** | Exclusive | 可读可写 | 与任何锁不兼容 |
| **共享锁(S锁)** | Shared | 只能读 | 与S锁兼容 |

**锁的相容矩阵**：
|      | S锁 | X锁 |
|------|-----|-----|
| S锁  | ✓   | ✗   |
| X锁  | ✗   | ✗   |

### 5.2 封锁协议

#### 一级封锁协议

> **内容**：事务修改数据前加**X锁**，直到**COMMIT/ROLLBACK**释放

**防止**：丢失更新

**允许**：脏读、不可重复读

#### 二级封锁协议

> **内容**：一级封锁协议 + 读取数据前加**S锁**，读完**立即**释放

**防止**：脏读

**允许**：不可重复读

#### 三级封锁协议

> **内容**：一级封锁协议 + 读取数据前加**S锁**，直到**COMMIT/ROLLBACK**释放

**防止**：脏读、不可重复读

**允许**：幻读

### 5.3 封锁协议对比

| 协议 | 防止丢失更新 | 防止脏读 | 防止不可重复读 | 防止幻读 |
|------|-------------|----------|----------------|----------|
| 一级 | ✓ | ✗ | ✗ | ✗ |
| 二级 | ✓ | ✓ | ✗ | ✗ |
| 三级 | ✓ | ✓ | ✓ | ✗ |

## 六、两段锁协议(2PL)

> **两段锁协议**：所有锁申请在解锁之前，扩展阶段只能加锁，收缩阶段只能解锁

### 6.1 协议内容

```
┌─────────────────────────────────────┐
│            扩展阶段                  │
│   获得所有锁，不释放任何锁           │
│   ● Lock(X)                         │
│   ● Lock(Y)                         │
│   ● Lock(Z)                         │
└──────────────────┬──────────────────┘
                   ↓
┌──────────────────┴──────────────────┐
│            收缩阶段                  │
│   释放所有锁，不获得任何新锁         │
│   ● Unlock(X)                       │
│   ● Unlock(Y)                       │
│   ● Unlock(Z)                       │
└─────────────────────────────────────┘
```

### 6.2 2PL的变种

| 类型 | 说明 |
|------|------|
| **严格2PL** | 保持X锁直到事务结束 |
| **强严格2PL** | 保持所有锁直到事务结束 |
| **锁粒度** | 封锁的数据单元大小 |

### 6.3 2PL的特点

- **可串行化**：保证并发执行可串行化
- **不会死锁**（但可能出现）
- **不会饿死**

## 七、隔离级别

### 7.1 SQL定义的隔离级别

| 级别 | 英文 | 脏读 | 不可重复读 | 幻读 |
|------|------|------|------------|------|
| **READ UNCOMMITTED** | 未提交读 | 可能 | 可能 | 可能 |
| **READ COMMITTED** | 已提交读 | 不可能 | 可能 | 可能 |
| **REPEATABLE READ** | 可重复读 | 不可能 | 不可能 | 可能 |
| **SERIALIZABLE** | 可串行化 | 不可能 | 不可能 | 不可能 |

### 7.2 设置隔离级别

```sql
SET TRANSACTION ISOLATION LEVEL READ COMMITTED;
```

### 7.3 各隔离级别说明

**READ UNCOMMITTED**：
- 可读取**未提交**的数据
- 可能发生脏读

**READ COMMITTED**：
- 只能读取**已提交**的数据
- 不可重复读可能发生

**REPEATABLE READ**：
- 同一事务中多次读取**同一数据**结果相同
- 幻读可能发生

**SERIALIZABLE**：
- 最高级别
- 完全可串行化
- 性能最低

## 八、封锁粒度

> **封锁粒度**：封锁的数据单元大小

### 8.1 粒度级别

| 粒度 | 级别 | 特点 |
|------|------|------|
| **属性** | 最小 | 控制精确，开销大 |
| **元组** | 小 | 并发度高，开销大 |
| **页/块** | 中 | 平衡 |
| **表** | 大 | 开销小，并发度低 |
| **数据库** | 最大 | 开销最小，并发度最低 |

### 8.2 多粒度封锁

**意向锁**：

| 锁类型 | 英文 | 说明 |
|--------|------|------|
| **IS** | 意向共享 | 准备获取S锁 |
| **IX** | 意向排他 | 准备获取X锁 |
| **SIX** | 共享意向排他 | 该表S锁+某些元组X锁 |

**锁兼容矩阵**：

|      | IS  | IX  | S   | X   |
|------|-----|-----|-----|-----|
| IS   | ✓   | ✓   | ✓   | ✗   |
| IX   | ✓   | ✓   | ✗   | ✗   |
| S    | ✓   | ✗   | ✓   | ✗   |
| X    | ✗   | ✗   | ✗   | ✗   |

## 九、死锁与活锁

### 9.1 死锁(Deadlock)

> **死锁**：两个或多个事务相互等待对方持有的锁

**死锁示例**：
```
T1: Lock(A) → 等待B
T2: Lock(B) → 等待A
```

### 9.2 死锁处理

| 方法 | 说明 |
|------|------|
| **预防死锁** | 破坏死锁条件 |
| **检测死锁** | 定期检测，等待图 |
| **解除死锁** | 选择牺牲事务回滚 |

### 9.3 死锁预防策略

| 策略 | 说明 |
|------|------|
| **一次性封锁** | 一次获取所有需要的锁 |
| **顺序封锁** | 按顺序封锁对象 |

## 十、日志与恢复

### 10.1 日志类型

| 类型 | 记录内容 | 用途 |
|------|----------|------|
| **UNDO日志** | 旧值 | 回滚未完成事务 |
| **REDO日志** | 新值 | 重做已提交事务 |
| **UNDO/REDO日志** | 两者 | 最灵活 |

### 10.2 日志记录格式

```
<Start T>              -- 事务开始
<T, X, old_v>          -- UNDO记录
<T, X, new_v>          -- REDO记录
<T, X, new_v, old_v>   -- UNDO/REDO记录
<Commit T>             -- 提交
<Abort T>              -- 中止
```

### 10.3 恢复原则

> **先写日志原则(WAL)**：日志必须先于数据写入磁盘

### 10.4 检查点(Checkpoint)

> **检查点**：将缓冲区内容写入磁盘，并记录检查点信息

**检查点作用**：
- 减少恢复时需要扫描的日志量
- 标记一个一致状态点

## 考点速记

### ACID特性（必须背诵）

| 特性 | 说明 |
|------|------|
| **A**tomicity | 原子性：全做或全不做 |
| **C**onsistency | 一致性：数据库状态一致 |
| **I**solation | 隔离性：并发事务互不干扰 |
| **D**urability | 持久性：提交后不丢失 |

### 事务状态

> 活动 → 部分提交 → 提交
>         ↘ 失败 → 中止 → 回滚

### 并发问题

| 问题 | 说明 |
|------|------|
| 丢失更新 | 修改被覆盖 |
| 脏读 | 读未提交数据 |
| 不可重复读 | 同一数据两次读不同 |
| 幻读 | 结果集变化 |

### 封锁协议对比

| 协议 | 规则 |
|------|------|
| 一级 | 修改前加X锁，直到结束 |
| 二级 | 一级 + 读前加S锁，立即释放 |
| 三级 | 一级 + 读前加S锁，直到结束 |

### 隔离级别

| 级别 | 脏读 | 不可重复读 | 幻读 |
|------|------|------------|------|
| RU | 可能 | 可能 | 可能 |
| RC | 不可能 | 可能 | 可能 |
| RR | 不可能 | 不可能 | 可能 |
| S | 不可能 | 不可能 | 不可能 |

### 锁类型

- **S锁(共享锁)**：可读，不可写
- **X锁(排他锁)**：可读，可写

### 两段锁协议

> 扩展阶段只加锁，收缩阶段只解锁

### 先写日志原则

> 日志必须先于数据写入磁盘
