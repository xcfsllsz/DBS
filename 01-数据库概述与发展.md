# 数据库概述与发展

## 一、数据库系统基本概念

### 1.1 什么是数据库

> **数据库(DB)**：以一定的组织形式存储在计算机内部的、能被多个用户共享的、结构化的、独立性的数据集合。

- **数据的组织结构**：层次、网状、关系型
- **数据独立性高**：物理独立性、逻辑独立性

### 1.2 数据库相关术语

| 术语 | 全称 | 说明 |
|------|------|------|
| DB | Data Base | 数据库 |
| DBMS | Data Base Management System | 数据库管理系统 |
| DBS | Data Base System | 数据库系统 |

### 1.3 数据库系统组成

```
┌─────────────────────────────────┐
│        数据库应用系统           │
│  ┌───────────────────────────┐  │
│  │       DBMS（核心）        │  │
│  └───────────────────────────┘  │
│  ┌───────────────────────────┐  │
│  │         数据库            │  │
│  └───────────────────────────┘  │
│  ┌───────────────────────────┐  │
│  │  DBA（数据库管理员）     │  │
│  └───────────────────────────┘  │
└─────────────────────────────────┘
```

## 三、数据库系统的特点

### 3.1 数据库相比文件系统的优势

**优点**：
- 提供程序与数据的数据独立性
- 支持程序与数据的结构化组织

**缺点（文件系统时代）**：
- 数据冗余度高
- 数据不一致
- 数据访问困难
- 无法支持并发访问
- 数据孤立（无关联性）
- 完整性问题
- 原子性问题
- 并发访问问题
- 安全性问题

### 3.2 数据库核心特点

1. **数据结构化**：数据以表（关系）的形式组织
2. **数据共享性高**：多个应用程序可共享同一数据
3. **数据独立性高**：物理独立性和逻辑独立性
4. **DBMS统一管理**：提供数据的安全性、完整性、并发控制、恢复功能

## 四、数据库系统结构

### 4.1 三级模式结构（从DBMS角度）

```
┌─────────────────┐
│    外模式        │ ← 用户视图
│ (External Schema)│
└────────┬────────┘
         │
┌────────┴────────┐
│    概念模式      │ ← 全局视图
│ (Conceptual)    │
└────────┬────────┘
         │
┌────────┴────────┐
│    内模式        │ ← 存储视图
│ (Internal Schema)│
└─────────────────┘
```

### 4.2 两级映射

- **外模式/概念模式映射**：外部视图 ↔ 全局逻辑结构
- **概念模式/内模式映射**：逻辑结构 ↔ 物理存储结构

### 4.3 两种角度的系统结构

| 角度 | 结构组成 |
|------|----------|
| DBMS角度 | 外模式、概念模式、内模式 |
| 用户角度 | 用户结构、概念结构、存储结构 |

## 五、数据库用户与DBA

### 5.1 数据库人员组成

- **DBA（数据库管理员）**：负责数据库的维护
- **系统管理员**：负责操作系统
- **应用开发人员**：负责应用程序
- **普通用户**：使用数据库

### 5.2 DBA职责（重点）

> 背诵内容

1. 决定数据库中的**信息内容和结构**
2. 决定数据库的**存储结构和存取策略**
3. 定义数据**安全性和完整性约束**
4. 监控数据库的**使用和运行**
5. 负责数据库的**改进和重组**

## 六、DBMS的工作过程

```
应用A → DBMS → 语法检查 → 权限检查
              ↓
         编译优化 → 执行计划
              ↓
         模式映射 → 存储管理
              ↓
         OS操作磁盘 → 返回结果
```

## 考点速记

### 必背概念

1. **DB**：以一定组织形式存储的数据集合
2. **DBMS**：管理和维护数据库的系统软件
3. **DBS**：包含DB、DBMS、DBA的完整系统
4. **数据独立性**：物理独立性（存储结构变化不影响逻辑）、逻辑独立性（逻辑结构变化不影响应用程序）

### 重要人物与贡献

| 人物 | 贡献 | 奖项年份 |
|------|------|----------|
| Codd | 关系模型 | 1981 |
| Bachman | 网络数据库IDS | 1973 |
| Gray | 事务处理 | 1998 |
| Stonebraker | Postgres | 2014 |

### 三级模式两极映射

- **外模式**：用户视图，多个
- **概念模式**：全局逻辑，一个
- **内模式**：存储结构，一个
- **映射**：外模式/概念模式映射、概念模式/内模式映射
